📝 Build Decisions

1. Firebase used to reduce backend effort in early stage
2. AWS used only in scaling due to cost concerns
3. Razorpay selected for UPI-first Indian market
4. Monolith preferred initially for faster debugging
5. Airtable used in no-backend case for speed
6. React avoided due to assignment constraint
7. India integrations prioritized
8. Complexity added only when required
